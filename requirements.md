# 요구사항 명세서

## 1. 개요
- 프로젝트명 : 여행 계획 관리 시스템 (Travel Planner)
- 개발 기간 : 2026-07-01 ~ 2026-07-15
- 개발 환경 : Windows 11, WSL2 (Ubuntu 24.04), Java 17, Spring Boot 3.5.4, MariaDB, Redis, React 19, Vite, MUI

## 2. 기능 요구사항

### 2.1 회원 관리
- 회원가입 (ID, 비밀번호, 닉네임, 이메일)
- 로그인 / 로그아웃 (JWT 기반)
- 내 정보 조회 및 수정
- 회원 탈퇴

### 2.2 여행 계획 관리 (Travel Plan)
- 여행 계획 생성 (지역, 시작일, 종료일, 목적지 키워드 목록)
- 여행 계획 목록 조회 (페이징, 정렬 가능)
- 여행 계획 상세 조회 (특정 ID 기준)
- 여행 계획 수정
- 여행 계획 삭제

### 2.3 목적지 관리 (Destination)
- 목적지 생성 (날짜, 시간, 키워드 리스트)
- 목적지 목록 조회
- 목적지 상세 조회
- 목적지 수정
- 목적지 삭제

### 2.4 AI 기반 여행 일정 추천
- 사용자가 입력한 지역, 날짜, 키워드, 예산, 동행 인원 등을 기반으로 OpenAI GPT-4o-mini 를 활용한 일정 자동 생성
- 생성된 일정은 여행 계획에 저장 (Destination 엔티티에 저장)

### 2.5 알림 및 캐시
- JWT 토큰 블랙리스트 관리를 위한 Redis 사용 (로그아웃 시 토큰 만료 처리)
- 자주 조회되는 여행지 정보 캐시 (선택 구현)

## 3. 비기능 요구사항

### 3.1 성능
- API 응답 시간 2초 이내 (평균)
- 동시 사용자 100명 기준으로 안정적 운영

### 3.2 보안
- 비밀번호는 BCrypt 암호화로 저장
- JWT 토큰은 Access Token (9시간) 및 Refresh Token (7일) 로테이션 적용
- Swagger UI 및 API 문서는 인증 없이 접근 가능
- CORS 정책: 프론트엔드 (http://localhost:5173) 만 허용

### 3.3 확장성
- 계층형 아키텍처 (Controller - Service - Repository)
- DTO 및 Entity 분리
- 인터페이스 기반 서비스 구현 (추후 구현 교체 용이)

### 3.4 호환성
- 브라우저: Chrome 최신버전, Edge, Firefox
- 반응형 웹 (모바일 지원)

## 4. 사용자 인터페이스 요구사항
- 로그인/회원가입 페이지
- 메인 페이지: 여행지 선택, 날짜 선택, AI 추천 생성 버튼
- 여행 계획 목록 페이지
- 여행 계획 상세 페이지 (일정표, 지도 연동 예정)
- 목적지 관리 모달 (추가/수정/삭제)

## 5. 제약조건
- JDK 17 이상 필수
- MariaDB 10.5 이상
- Redis 6 이상
- Node.js 18 + npm 9 이상
- 배포 방식: 로컬 실행 (Gradle bootRun, Vite dev server) 또는 Docker-compose (선택)
